# Proposed upgraded system model: UPAIR-5G

## Name
**UPAIR-5G** = **U**niversal **P**rompted **A**xial **I**n-context **R**eceiver for **5G NR PUSCH**

## Why this is aligned with the paper philosophy
The original paper is built around:
- preserving **the speed of wireless**
- avoiding heavy offline adaptation at TTI time
- telling the network **what we already know**
- learning only the missing part from the current radio context

UPAIR-5G preserves that philosophy exactly:
- **No per-slot gradient update at inference**
- Current-slot **DMRS pilots act as the in-context prompt**
- A **model-driven LS estimate** is the known physics prior
- A lightweight **AI refiner** learns only the residual mismatch

## Core technical idea
The proposed receiver replaces the default LS estimator inside the Sionna PUSCH receiver with a new estimator:

### 1) Physics-informed front-end
- Standard **LS DMRS channel estimate**
- Standard PUSCH-compatible resource-grid structure
- Keeps 3GPP NR compliance

### 2) Slot-conditioned AI adaptation
The AI module reads:
- LS estimate
- received PUSCH resource grid
- pilot mask
- noise level

Then it forms a **DMRS-conditioned prompt vector** from the current slot only.

This means adaptation is **instant and feed-forward**, not slow gradient-based online training.

### 3) Axial attention over the time-frequency grid
The refiner uses:
- attention along the **frequency axis**
- attention along the **time axis**
- local depthwise convolution

This gives the receiver a strong inductive bias for **doubly selective channels** while staying computationally lighter than full 2D attention.

### 4) Uncertainty-aware output
The network predicts:
- a **residual correction** to the LS estimate
- a **refined error-variance map**

The downstream detector remains **standard LMMSE**, which makes comparison clean and publishable.

## Why this scenario should favor the proposal over LS+LMMSE
The package targets a scenario where the Sionna default receiver is weak:
- **5G NR PUSCH**
- **CDL-C NLoS**
- **high mobility**
- **single DMRS symbol / sparse pilots**
- **residual common phase drift across OFDM symbols**

Why this is good:
- LS + linear interpolation struggles when the channel is strongly time-varying and when extra phase drift exists between DMRS and data symbols
- the prompt-conditioned axial refiner can exploit:
  - global time-frequency structure
  - current-slot pilot context
  - cross-symbol consistency
  - multi-antenna observation structure

## What is still intentionally classical
To preserve trustworthiness and a clean ablation:
- modulator = standard PUSCH
- coding = standard PUSCH
- detector = standard LMMSE
- decoder = standard TB decoder

So the gain, if observed, comes from the **AI-native channel-estimation stage**.

## Why this is innovative enough for a strong paper direction
The upgrade combines:
- **in-context adaptation** from modern AI
- **prompt conditioning**
- **axial attention**
- **uncertainty-aware model-driven estimation**
- strict **5G NR PUSCH compatibility**

That combination is much more AI-native than the original reservoir-style design while still respecting the “speed of wireless” principle.
